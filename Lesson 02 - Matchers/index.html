Matchers

In this lesson I learnt about the various matchers available to Jasmine. Matchers are used to implement a comparison between the actual value and expected value of an expectation.

The 'toBe()' matcher

This matcher tests where the actual value equates(===) to the expected value passed to the matcher. The is not to be confused with the 'toEqual()' matcher which tests for deeper equality such as between objects

  1. it('should add numbers to total', () => {
  2.   const calculator = new Calculator();
  3.   calculator.total = 40;
  4.   calculator.add(5);
  5.  
  6.   // 'toBe()' matchers expects the actual value to be === to the expected value.
  7.   expect(calculator.total).toBe(45);
  8. });
The 'toEqual()' matcher

This matcher is used to compare objects/arrays passed for deeper equality comparison since objects/arrays are reference types and will fail if compared using the toBe() matcher.

  1. it('should have a constructor', () => {
  2.   const calculator = new Calculator();
  3.   const calculator_two = new Calculator();
  4.  
  5.   /**
  6.   * Here we use the toEqual() matcher to compare the
  7.   * objects passed for deeper equality comparison since
  8.   * objects are reference types and will fail if compared
  9.   * using the toBe() matcher
  10.   */
  11.   expect(calculator).toEqual(calculator_two);
  12. });
The 'toBeTruthy()' matcher

This matcher expects the actual value to be truthy. Truthy values are values that translate to a boolean result of true. The opposite happens for a falsy value where we use the toBeFalsy() matcher.

  1. it('can be instantiated', () => {
  2.   const calculator = new Calculator();
  3.  
  4.   /**
  5.   * This expects the actual value to be truthy.
  6.   */
  7.   expect(calculator).toBeTruthy();
  8. });
The 'not' matcher

We use the 'not' reference to negate a matcher. This inverts the behaviour of the matcher when comparing the actual value to the expected value. 'not' can be used together with any matcher.

  1. it('instatiates a unique object', () => {
  2.   const calculator = new Calculator();
  3.   const calculator_two = new Calculator();
  4.  
  5.   /**
  6.   * inverts the behaviour of any matcher
  7.   */
  8.   expect(calculator).not.toBe(calculator_two);
  9. });
The 'toBeDefined' & 'toBeUndefined' matchers

The 'toBeUndefined' matcher determines whether the value in our expectation equates to undefined. The 'toBeDefined' matcher tests the opposite of the 'toBeUndefined' matcher. 

  1. it('has common operations', () => {
  2.   const calculator = new Calculator();
  3.  
  4.   /**
  5.   * tests if the value is undefined
  6.   */
  7.   expect(calculator.add).not.toBeUndefined();
  8.   expect(calculator.subtract).not.toBeUndefined();
  9.   expect(calculator.multiply).not.toBeUndefined();
  10.  
  11.   /**
  12.   * this is equivalent to '.not.toBeUndefined'
  13.   */
  14.   expect(calculator.divide).toBeDefined();
  15. });
The 'toContain' matcher

This matcher is used to test whether the value passed contains a specified substring. It can also be used to test whether an element exists in an object or array. 

  1. it('has common operations', () => {
  2.   const calculator = new Calculator();
  3.  
  4.  /**
  5.  * checks for a substring
  6.  */
  7.  expect(calculator.constructor.name).toContain('Calc');
  8. });
The 'toBeNull' matcher

This matcher tests whether the actual value is equal to null. 

  1. it('can overwrite total', () => {
  2.   const calculator = new Calculator();
  3.  
  4.   // let overwite total to null
  5.   calculator.total = null;
  6.  
  7.   /**
  8.   * tests whether the actual value is null
  9.   */
  10.   expect(calculator.total).toBeNull();
  11. });
The 'toBeNaN' matcher

The matcher 'toBeNaN' is used to test whether the value passed is not a number(NaN). 

  1. it('does not handle NaN', () => {
  2.   const calculator  = new Calculator();
  3.  
  4.   calculator.total = 20;
  5.   calculator.multiply('a');
  6.  
  7.   /**
  8.   * test whether the value passed is not a number.
  9.   */
  10.   expect(calculator.total).toBeNaN();
  11. });
The 'toThrow' & 'toThrowError' matchers

The matcher 'toThrow' is used to test that the callback passed throws an exception. This can used to test wether a function handles exceptions. The 'toThrowError' matcher tests what kind of exception is thrown by passing the expected exception class. This matcher also accepts a message as a second parameter which checks wether the message thrown by the exception class matches. 

  1. it('handles divide by zero', () => {
  2.   const calculator  = new Calculator();
  3.  
  4.   /**
  5.   * tests that the callback passed throws an exception.
  6.   */
  7.   expect(() => calculator.divide(0)).toThrow();
  8.  
  9.   /**
  10.   * tests what kind of exception is thrown by passing the expected exception class
  11.   */
  12.   expect(() => calculator.divide(0)).toThrowError(Error);
  13.  
  14.   /**
  15.   * we also test the message of the exception thrown matches with our expected value
  16.   */
  17.   expect(() => calculator.divide(0)).toThrowError(Error, 'Cannot divide by zero');
  18. });
The 'toMatch' matcher

The matcher 'toMatch' is used to test whether the expected value matches a regular expression. 

  1. it('returns a positive or negative total', () => {
  2.   const calculator  = new Calculator();
  3.   calculator.total = 50;
  4.  
  5.   expect(calculator.add(20)).toBe(70);
  6.  
  7.   /**
  8.   * The below regex checks for positive and negative digits
  9.   */
  10.   expect(calculator.total).toMatch(/-?\d+/);
  11.   // This is a simpler regular expression matching the type
  12.   expect(typeof calculator.total).toMatch('number');
  13. });
jasmine.anything()

jasmine.anything() is used by any matcher that uses equality such as toEqual, toContain and more. It tests any value as long as it is not null or undefined. 

  1. it('returns a positive or negative total', () => {
  2.   const calculator  = new Calculator();
  3.   calculator.total = 50;
  4.  
  5.   expect(calculator.add(20)).toBe(70);
  6.  
  7.   /**
  8.   * tests any value as long as it is not null or undefined
  9.   */
  10.   expect(calculator.total).toEqual(jasmine.anything());
  11. });
